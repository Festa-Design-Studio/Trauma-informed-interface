Login Template Documentation

Usage

This template provides a trauma-informed login page that balances security with accessibility and emotional safety.

Key Components

  • Email and password login form with appropriate validation
  • Password visibility toggle option
  • OAuth provider options (Google, Microsoft)
  • Password reset link
  • Account creation link
  • Emergency exit button
  • Supportive visual elements and messaging

Implementation Notes

  • Replace placeholder action URLs with actual authentication endpoints
  • Implement server-side validation for form submissions
  • Configure OAuth provider integrations
  • Replace the welcome illustration with an appropriate calming image
  • Consider implementing rate limiting to prevent brute force attacks

Trauma-Informed Considerations

  • Password visibility toggle helps users who experience anxiety about typos
  • OAuth options provide alternative authentication methods
  • Supportive messaging reduces potential login anxiety
  • Clear help section with support contact
  • Visual design uses calming colors and adequate spacing
  • Emergency exit button provides immediate way out if needed
  • Form elements have adequate sizing for users with motor control difficulties
  • Error states (to be implemented server-side) should be clear but gentle
