Usage Guidelines

Usage:

  • For collecting single-line text input from users
  • Common for names, emails, short answers
  • Always pair with descriptive labels

Accessibility:

  • Maintain adequate height for touch targets
  • Use proper label associations with 'for' and 'id'
  • Include descriptive placeholder text
  • Error messages should be associated with inputs using aria-describedby

Trauma-informed considerations:

  • Clear labeling reduces uncertainty and anxiety
  • Gentle transitions for focus states avoid jarring changes
  • Generous padding creates visual breathing room
  • Explicitly marking optional fields gives users control
  • Error messages use supportive, non-judgmental language
  • Soft color palette avoids triggering stress responses
  • Rounded corners create a softer, less threatening appearance
