Usage Guidelines

Usage:

  • For selecting one option from a list of mutually exclusive choices
  • For preference selections where only one option can be chosen
  • For branching paths in content or learning journeys

Accessibility:

  • Group related radio buttons with fieldset and legend
  • Use proper label associations with 'for' and 'id'
  • Maintain adequate size for touch targets
  • Ensure focus states are clearly visible
  • Card-style variants should use the "peer" classes for accessible styling

Trauma-informed considerations:

  • Clear descriptions help users make informed choices
  • Visual card style options create more space and reduce cognitive load
  • Labels are descriptive and non-judgmental
  • Content intensity options let users control their experience
  • Options are presented equally without suggesting a "correct" choice
  • Subtle but clear visual feedback for selected state
  • Smooth transitions prevent jarring visual changes
