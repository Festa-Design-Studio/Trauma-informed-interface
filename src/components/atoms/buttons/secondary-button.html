Secondary Button Component

The secondary button is used for important but non-primary actions. It provides a calm, clear alternative to the primary button.

Default Secondary Button

<button class="border-2 border-chrysocolla-green-DEFAULT text-chrysocolla-green-DEFAULT px-md py-sm rounded-md hover:bg-chrysocolla-green-50 transition-all duration-DEFAULT font-montserrat font-medium">Back</button>

Use for secondary actions, such as going back, canceling, or alternative options. The calm green color and border create a sense of safety and clear boundaries.

Full-width Secondary Button

<button class="w-full border-2 border-chrysocolla-green-DEFAULT ...">Learn More</button>

Use full-width for secondary actions in forms, cards, or modals to maintain alignment and provide a clear, accessible target.

Secondary Button with Icon

<button class="... flex items-center gap-2">Back <svg ... /></button>

Use an icon to clarify the action (e.g., back, download). Icons reinforce meaning and support users with lower literacy or language barriers.

Small Secondary Button

<button class="border-2 border-chrysocolla-green-DEFAULT ... py-xs px-sm ... text-small">Save</button>

Use for compact layouts or toolbars where space is limited. Still provides clear feedback and accessibility.

Disabled Secondary Button

<button class="border-2 border-chrysocolla-green-200 ... opacity-60 cursor-not-allowed" disabled>Back</button>

Use to indicate an unavailable or inactive action. The faded color and cursor communicate that the action is not possible.

Loading State Secondary Button

<button class="... flex items-center ..." disabled><svg class="animate-spin ..." ... />Loading...</button>

Use when an action is processing. The spinner and disabled state provide clear, non-alarming feedback that the system is working.

Secondary Button with Left Icon

<button class="... flex items-center gap-2"><svg ... />Download</button>

Use a left icon for actions like download or back. Icons help users quickly recognize the action, supporting accessibility and clarity.

Buttons on Colored Backgrounds & Dark Mode

On Colored Backgrounds

<button class="bg-selenite-white-50 text-azurite-blue-700 border ...">Secondary Action</button>

On colored backgrounds, use a white background and blue text for the secondary button to ensure contrast and clarity.

In Dark Mode

<button class="bg-grey-800 text-selenite-white-50 ... dark:bg-selenite-white-50 dark:text-azurite-blue-900">Secondary Action</button>

For dark mode, use a dark background and white text, or invert for accessibility using dark: classes.

Accessibility

  • Maintain at least 4.5:1 contrast for text and backgrounds.
  • Ensure focus and hover states are clear and visible.
  • Use border and color changes for clear secondary action indication.

Usage Guidelines

Usage:

  • Use for important actions that are not the primary focus, such as cancel, back, or alternative choices.
  • Place secondary buttons to the left of primary buttons in left-to-right layouts for clear hierarchy.
  • Use icon variants to clarify direction or reinforce meaning (e.g., arrows for back, download icons for downloads).
  • Use the small variant for toolbars or compact layouts.
  • Use the disabled and loading states to indicate unavailable or processing actions, reducing user frustration.

Accessibility:

  • Ensure sufficient color contrast between button, border, and text.
  • Maintain a minimum touch target size (44x44px).
  • Use aria-labels for icon-only or ambiguous buttons.
  • Provide visible focus states for keyboard navigation.
  • Avoid disabling buttons without explanation; if disabled, provide context.

Trauma-informed Considerations:

  • The calm green color and clear border are chosen to avoid alarming users and to create a sense of safety and boundaries.
  • Gentle color transitions on hover provide predictable, calming feedback and reinforce user control.
  • Consistent button placement and sizing reduce cognitive load and decision fatigue.
  • Disabled and loading states prevent confusion and help users feel safe to pause or wait.
  • Icons support users with lower literacy or language barriers.
  • Soft, rounded corners and calm color palette avoid triggering stress responses.
