Tertiary Button Component

The tertiary button is used for optional, low-emphasis actions. It is visually minimal and supports user autonomy.

Default Tertiary Button

<button class="font-montserrat text-small text-chrysocolla-green-DEFAULT hover:text-chrysocolla-green-600 transition-all duration-DEFAULT underline">Continue Learning</button>

Use for optional, low-emphasis actions such as “Continue Learning” or “Learn more”. Minimal styling reduces pressure and supports user autonomy.

Full-width Tertiary Button

<button class="w-full font-montserrat text-small text-chrysocolla-green-DEFAULT ... underline text-center">View All Resources</button>

Use full-width for tertiary actions in cards or forms to create a clear, accessible target and maintain alignment with other button types.

Tertiary Button with Icon

<button class="... flex items-center gap-2">Learn more <svg ... /></button>

Use an icon to clarify the action. Icons reinforce meaning and support users with lower literacy or language barriers.

Small Tertiary Button

<button class="font-montserrat text-caption text-chrysocolla-green-DEFAULT ... px-xs py-0.5 ... underline">Details</button>

Use for compact layouts or inline actions where space is limited.

Disabled Tertiary Button

<button class="font-montserrat text-small text-grey-400 underline cursor-not-allowed opacity-60" disabled>Continue Learning</button>

Use to indicate an unavailable or inactive action. The faded color and cursor communicate that the action is not possible.

Loading State Tertiary Button

<button class="... flex items-center ..." disabled><svg class="animate-spin ..." ... />Loading...</button>

Use when an action is processing. The spinner and disabled state provide clear, non-alarming feedback that the system is working.

Tertiary Button with Left Icon

<button class="... flex items-center gap-2"><svg ... />Back to Overview</button>

Use a left icon for actions like “Back” or “Return”. Icons help users quickly recognize the action, supporting accessibility and clarity.

Buttons on Colored Backgrounds & Dark Mode

On Colored Backgrounds

<button class="bg-selenite-white-50 text-azurite-blue-700 border ...">Tertiary Action</button>

On colored backgrounds, use a white background and blue text for the tertiary button to ensure contrast and clarity.

In Dark Mode

<button class="bg-grey-800 text-selenite-white-50 ... dark:bg-selenite-white-50 dark:text-azurite-blue-900">Tertiary Action</button>

For dark mode, use a dark background and white text, or invert for accessibility using dark: classes.

Accessibility

  • Maintain at least 4.5:1 contrast for text and backgrounds.
  • Ensure focus and hover states are clear and visible.
  • Use border and color changes for clear tertiary action indication.

Usage Guidelines

Usage:

  • Use for optional, low-emphasis actions, such as "Learn more" or "Continue Learning" links.
  • Use icon variants to clarify direction or reinforce meaning.
  • Use the small variant for compact layouts or inline actions.
  • Use the disabled and loading states to indicate unavailable or processing actions.

Accessibility:

  • Ensure sufficient color contrast for text and underline.
  • Maintain a minimum touch target size (44x44px) where possible.
  • Use aria-labels for icon-only or ambiguous buttons.
  • Provide visible focus states for keyboard navigation.

Trauma-informed Considerations:

  • Minimal visual presence reduces cognitive load and pressure.
  • Gentle color transitions and underlines reinforce interactivity without demanding action.
  • Disabled and loading states prevent confusion and help users feel safe to pause or wait.
  • Icons support users with lower literacy or language barriers.
  • Consistent sizing and placement reduce cognitive load.
