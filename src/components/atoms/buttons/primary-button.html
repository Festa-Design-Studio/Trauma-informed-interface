Primary Button Component

The primary button is used for main actions and calls to action. It is designed for clarity, accessibility, and trauma-informed interaction.

Default Primary Button

<button class="bg-chrysocolla-green-DEFAULT text-selenite-white-50 px-md py-sm rounded-md hover:bg-chrysocolla-green-600 transition-all duration-DEFAULT font-montserrat font-medium">Continue</button>

Use for main actions such as submitting forms or continuing to the next step. The green color signals safety and progress, and the gentle hover transition provides reassuring feedback.

Full-width Primary Button

<button class="w-full bg-chrysocolla-green-DEFAULT ...">Complete Registration</button>

Use full-width for forms and cards to create a clear, accessible target and reduce decision fatigue. This is especially helpful for trauma-affected users who benefit from clear, prominent actions.

Primary Button with Icon

<button class="... flex items-center gap-2">Continue <svg ... /></button>

Use an icon to clarify the action (e.g., next, download). Icons reinforce meaning and support users with lower literacy or language barriers.

Small Primary Button

<button class="bg-chrysocolla-green-DEFAULT ... py-xs px-sm ... text-small">Save</button>

Use for secondary actions in toolbars or tight spaces. Still provides clear feedback and accessibility.

Disabled Primary Button

<button class="bg-chrysocolla-green-200 ... opacity-60 cursor-not-allowed" disabled>Continue</button>

Use for unavailable actions. The faded color and cursor indicate the action is not currently possible, reducing frustration and confusion.

Loading State Primary Button

<button class="... flex items-center ..." disabled><svg class="animate-spin ..." ... />Loading...</button>

Use when an action is processing. The spinner and disabled state provide clear, non-alarming feedback that the system is working.

Primary Button with Left Icon

<button class="... flex items-center gap-2"><svg ... />Download Toolkit</button>

Use a left icon for actions like download or back. Icons help users quickly recognize the action, supporting accessibility and clarity.

Usage Guidelines

Usage:

  • Use for main actions and calls to action, such as submitting forms, continuing to the next step, or starting a course.
  • Use icon variants to clarify direction or reinforce meaning (e.g., arrows for next, download icons for downloads).
  • Use the small variant for toolbars or compact layouts.
  • Use the disabled and loading states to indicate unavailable or processing actions, reducing user frustration.

Accessibility:

  • Ensure sufficient color contrast between button and text.
  • Maintain a minimum touch target size (44x44px).
  • Use aria-labels for icon-only or ambiguous buttons.
  • Provide visible focus states for keyboard navigation.

Trauma-informed Considerations:

  • Gentle color transitions on hover provide predictable, calming feedback and reinforce user control.
  • Consistent button placement and sizing reduce cognitive load and decision fatigue.
  • Disabled and loading states prevent confusion and help users feel safe to pause or wait.
  • Icons support users with lower literacy or language barriers.
  • Soft, rounded corners and calm color palette avoid triggering stress responses.

Buttons on Colored Backgrounds & Dark Mode

On Colored Backgrounds

<button class="bg-selenite-white-50 text-chrysocolla-green-DEFAULT ...">Continue</button>

When placing a primary button on a dark or colored background, use a light button background (e.g., bg-selenite-white-50) and colored text for contrast and accessibility.

In Dark Mode

<button class="bg-chrysocolla-green-600 text-selenite-white-50 ... dark:bg-chrysocolla-green-400 dark:text-grey-900">Continue</button>

For dark mode, use a more saturated button color (e.g., bg-chrysocolla-green-600) and ensure text remains readable. Use dark: classes for Tailwind or custom CSS for other frameworks.

Accessibility

  • Always check color contrast (WCAG AA: 4.5:1 for text).
  • Test button focus and hover states on all backgrounds.
  • Use visible outlines or shadows for focus in dark mode.
