Usage Guidelines

Usage:

  • To show progress through courses, modules, or forms
  • To provide orientation within multi-step processes
  • To visualize completion status of tasks
  • To create interactive navigation for course journeys

Variations:

  • Standard linear progress bar for simple progress indication
  • Step indicators for discrete stages
  • Circle progress for compact displays
  • Interactive segments for navigation through completed portions

Accessibility:

  • Include text alternatives for screen readers
  • Use sufficient color contrast (minimum 3:1 for UI elements)
  • Make interactive progress elements keyboard accessible
  • Use aria-label for descriptive information on interactive elements

Trauma-informed Considerations:

  • Progress visualization creates predictability and reduces anxiety
  • Clear indication of current location helps maintain orientation
  • Calm color palette avoids triggering stress responses
  • Rounded shapes create softer, less threatening appearance
  • Completed segments use green to reinforce positive accomplishment
  • Current segments use blue to indicate neutral, non-judgmental status
  • Text descriptions provide reinforcing feedback on progress
  • Step indicators break large tasks into manageable segments
  • Interactive elements allow users to revisit completed content when needed
