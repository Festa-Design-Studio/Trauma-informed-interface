Usage Guidelines

Usage:

  • For card actions and secondary information
  • Typically contains buttons, links, or metadata
  • Placed at the bottom of the card
  • Can include tags, progress indicators, or timestamps

Variations:

  • Single button for primary action
  • Multiple buttons for action choices
  • Text and button combinations for context with action
  • Tag collections for categorization
  • Progress with action for learning content

Accessibility:

  • Ensure buttons have descriptive aria-labels
  • Maintain sufficient contrast for text and buttons
  • Make sure interactive elements have adequate size
  • Provide keyboard focus states for all interactive elements
  • Stack elements vertically on small screens for touch targets

Trauma-informed Considerations:

  • Clear separation from main content with subtle border
  • Consistent padding creates visual stability
  • Non-pressuring action button language
  • Progress indicators provide reinforcing feedback
  • Multiple options when appropriate to provide user control
  • Tags help users find relevant content based on needs
  • Responsive design adapts to different screen sizes
  • Adequate spacing between interactive elements prevents mis-taps
  • Soft color palette avoids triggering stress responses
